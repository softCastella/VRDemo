# VR 화학물질 안전 프로젝트 손 포즈 시스템 설명

이 문서는 VR에서 손으로 소화기, 화학물질 소품, 버튼 등을 자연스럽게 잡고 누르는 시스템을 이해하기 위한 정리 문서이다.

---

## 목표

우리 프로젝트에서 필요한 것은 다음과 같다.

```text
평소에는 VR 컨트롤러 손 애니메이션이 보임
물체를 잡으면 그 물체에 맞는 손 포즈로 바뀜
물체를 놓으면 다시 기존 손 애니메이션으로 돌아감
```

예를 들어 소화기를 잡을 때는 그냥 주먹 모양이 아니라, 소화기 손잡이에 맞는 손 모양이 나와야 한다.

---

## 선생님이 원하는 핵심

선생님이 원하는 것은 단순히 손 포즈 데이터를 따로 만드는 것이 아니라, **Unity Editor 안에서 사용할 수 있는 제작 툴**을 만드는 것이다.

즉, 사람이 매번 파일을 직접 만들고 오브젝트를 손으로 연결하는 것이 아니라, 에디터 툴에서 버튼을 눌러 다음 작업이 자동으로 되게 하는 것이다.

```text
1. Unity Editor에서 손 모델을 원하는 모양으로 포징한다
2. 에디터 툴의 저장 버튼을 누른다
3. 현재 손목 위치와 손가락 포즈가 HandPose 데이터로 저장된다
4. 선택한 물체에 AttachPoint가 생성되거나 연결된다
5. 물체에 고스트 핸드가 자동으로 생성된다
6. 물체 프리팹에 HandPose 데이터가 연결된다
```

한마디로:

> 포징해서 버튼을 누르면, 포즈 데이터 저장과 고스트 핸드 생성까지 자동으로 끝나는 툴이 필요하다.

---

## 에디터 툴이 해야 할 일

Unity Editor용 툴은 대략 이런 기능을 가져야 한다.

```text
선택한 손 모델 읽기
선택한 물체 읽기
손목 위치와 회전 저장
손가락 뼈들의 회전값 저장
HandPose ScriptableObject 생성
GhostHand 프리팹 또는 자식 오브젝트 생성
AttachPoint 생성 또는 위치 저장
물체에 HandPose 참조 연결
```

예를 들어 툴 창에는 이런 버튼이 있을 수 있다.

```text
[현재 손 포즈 저장]
[고스트 핸드 생성]
[선택한 물체에 포즈 적용]
[프리팹 저장]
```

더 편하게 만들면 버튼 하나로 합칠 수도 있다.

```text
[포즈 저장 + 고스트 핸드 생성]
```

이 버튼을 누르면 현재 포징된 손 상태를 읽어서, 필요한 데이터와 오브젝트를 자동으로 만들어준다.

---

## 기본 제작 흐름

선생님이 말한 흐름은 다음과 같이 이해하면 된다.

```text
1. 핸드 비주얼라이저로 손 포즈를 만든다
2. 그 손 포즈를 애니메이션 클립 또는 HandPose 데이터로 저장한다
3. 손 모델과 포즈를 프리팹으로 만든다
4. 물체 프리팹에 고스트 핸드와 AttachPoint를 붙인다
5. 실제 플레이 중 물체를 잡으면 저장된 포즈를 실제 손에 적용한다
6. 물체를 놓으면 다시 기본 손 애니메이션으로 돌아간다
```

기존 방식으로 하면 이 과정을 사람이 하나하나 해야 한다.

하지만 에디터 툴을 만들면 다음처럼 줄일 수 있다.

```text
손 포징
→ 버튼 클릭
→ HandPose 데이터 자동 저장
→ GhostHand 자동 생성
→ AttachPoint 자동 연결
→ 물체 프리팹에 포즈 자동 연결
```

---

## 고스트 핸드와 클립의 차이

고스트 핸드는 클립 그 자체가 아니다.

```text
고스트 핸드 = 손 모양을 미리 보여주는 손 모델
클립 = 손가락이 어떻게 굽어 있는지 저장한 애니메이션 데이터
```

즉, 고스트 핸드는 눈에 보이는 손 모델이고, 클립은 그 손 모델을 특정 자세로 만드는 데이터이다.

예시:

```text
GhostHand_FireExtinguisher
└─ HandPose_FireExtinguisher.anim 적용
```

또는 ScriptableObject 방식이면 이렇게 볼 수 있다.

```text
GhostHand_FireExtinguisher
└─ HandPose_FireExtinguisher.asset의 포즈값 적용
```

## AttachPoint란?

AttachPoint는 물체를 잡았을 때 손목이 붙을 위치이다.

예를 들어 소화기를 잡을 때 손이 손잡이에 정확히 붙어야 한다.

```text
FireExtinguisher
├─ Mesh
├─ Collider
├─ XR Grab Interactable
├─ AttachPoint
└─ GhostHand
```

AttachPoint는 다음 역할을 한다.

```text
손목 위치 기준
손목 회전 기준
물체를 잡았을 때 손이 붙는 기준점
```

---

## 손목 위치와 손가락 변화를 둘 다 저장해야 하는 이유

손 포즈를 저장할 때는 손가락 모양만 저장하면 부족하다.

반드시 다음 두 가지를 함께 저장해야 한다.

```text
1. 손목 위치와 회전
2. 손가락 뼈들의 구체적인 회전값
```

손가락 모양만 저장하면 이런 문제가 생길 수 있다.

```text
손가락은 잘 굽어 있음
하지만 손 전체 위치가 물체와 맞지 않음
```

그래서 손목 위치도 함께 맞춰야 한다.

```text
손목 위치가 AttachPoint에 맞음
손가락이 물체 모양에 맞게 굽음
결과적으로 자연스럽게 잡은 것처럼 보임
```

---

## Hand Pose 데이터에 들어갈 내용

하나의 손 포즈 데이터에는 대략 이런 정보가 필요하다.

```text
HandPose_FireExtinguisher
├─ 손목 위치 보정값
├─ 손목 회전 보정값
├─ 엄지 뼈 회전값
├─ 검지 뼈 회전값
├─ 중지 뼈 회전값
├─ 약지 뼈 회전값
└─ 새끼손가락 뼈 회전값
```

이 데이터를 ScriptableObject로 저장하면 물체마다 쉽게 재사용할 수 있다.

예시:

```text
HandPose_FireExtinguisher.asset
HandPose_ChemicalBottle.asset
HandPose_Beaker.asset
HandPose_Button.asset
```

## 잡을 때의 동작

플레이어가 물체를 잡으면 다음 순서로 동작한다.

```text
1. 잡은 물체에서 HandPose 데이터를 찾음
2. 손목을 물체의 AttachPoint 위치로 이동시킴
3. 기존 기본 손 애니메이션을 잠시 끔
4. 저장된 손가락 포즈를 실제 손에 적용함
```

여기서 "기존 애니메이션을 없앤다"는 말은 삭제한다는 뜻이 아니다.

정확히는:

```text
잡고 있는 동안 기본 그립 애니메이션 대신
물체 전용 손 포즈를 우선 적용한다
```

라는 뜻이다.

---

## 놓을 때의 동작

플레이어가 물체를 놓으면 다음 순서로 동작한다.

```text
1. 물체 전용 HandPose 적용을 해제함
2. 손목이 다시 컨트롤러 위치를 따라감
3. 손가락이 다시 기본 애니메이션을 따라감
```

즉, 물체를 놓으면 다시 평소처럼 손이 움직여야 한다.

---

## 최종 구조 예시

잡을 수 있는 물체 프리팹은 대략 이런 구조가 된다.

```text
GrabObject
├─ Mesh
├─ Collider
├─ XR Grab Interactable
├─ AttachPoint
├─ GhostHand
└─ HandPose 데이터 연결
```

실제 손 시스템은 이렇게 동작한다.

```text
평상시
→ 컨트롤러를 따라가는 기본 손 애니메이션

물체 잡음
→ AttachPoint에 손목을 맞춤
→ 물체 전용 손 포즈 적용

물체 놓음
→ 물체 전용 손 포즈 해제
→ 기본 손 애니메이션 복귀
```

---

## 한 문장 요약

물체마다 손목 위치와 손가락 모양을 미리 저장해두고, Unity Editor 툴에서 버튼 한 번으로 HandPose 데이터 저장과 GhostHand 생성을 자동화한 뒤, 플레이어가 그 물체를 잡으면 실제 손이 저장된 모습으로 바뀌고 놓으면 다시 원래 VR 손 애니메이션으로 돌아가는 시스템이다.
